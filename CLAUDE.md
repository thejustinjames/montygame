# MontyGame — Project Guide for Claude

MontyGame is an **early-stage game ideation repository**. It captures concepts,
planning, and reference material as the game takes shape. There is no
application or code to build yet — the focus is ideas, design, and research.

## Repository structure

| Folder | Purpose |
| --- | --- |
| `Docs/` | Design documents, specs, and written concepts. |
| `Planning/` | Roadmaps, task lists, and milestone planning. |
| `Research/` | Reference imagery, mood boards, and inspiration (sci-fi, robots, dinosaurs, space themes so far). |

## Key files

- `README.md` — public overview of the repo.
- `LICENSE` — **All Rights Reserved**. Proprietary; the owner retains full rights and may commercialize. Do not add permissive license headers or invite reuse without the owner's say-so.
- `PRIVACY.md` — privacy policy (no data collected at this stage).
- `.gitignore` — excludes `.DS_Store`, editor cruft, and `.claude/settings.local.json`.

## Working conventions

- **Repo:** public, hosted at https://github.com/thejustinjames/montygame (`origin`, `main` branch).
- **Empty folders** are kept in git via `.gitkeep` files — don't delete those.
- **Commit/push** only when asked. When committing, use clear messages describing the ideation change.
- This is a **creative project** — when helping with design docs, planning, or naming, lean into game-design thinking, not just software engineering.
- If the project grows into actual game code, update this file and `PRIVACY.md` to match.
